H2E: HER2NI Evaluation

H2E is the official implementation suite that operationalizes HER2NI as an interaction-trajectory evaluator—a scoring and analysis layer that tracks stability and drift over multi-turn exchanges (human–AI, agent–agent, or mixed).

Interaction Telemetry (Cs, Ss, Hs) + Hs(t)
Drift & Collapse Detection Model-Agnostic Early Warning Signals
What H2E Produces
  • Per-turn metrics: Cs, Ss, Hs (optionally vector-valued internally).
  • Trajectory curves: Hs(t) and derived features (volatility, trend, collapse events).
  • Event flags: drift onset, contradiction clustering, instability spikes, recovery signatures.
  • Audit artifacts: JSON/CBOR telemetry packets + reproducible summary reports.
What H2E Does Not Do
  • Not a diagnostic or clinical tool. No health inference, no patient claims.
  • Not a model or training method by itself. H2E is telemetry; it can be used downstream, but it does not prescribe weights.
  • Not a truth-judger. It measures stability and drift, not correctness or ideology.
  • Not an enforcement system. Any intervention remains the user’s or institution’s responsibility.

H2E Signal Model
Trajectory (not point outputs) Relational coherence Drift geometry Collapse / recovery markers Model-agnostic

H2E evaluates interaction stability as a trajectory in observable space. It is compatible with human–AI dialogue, multi-agent systems, and networked tool-using agents. The core output is a time series Hs(t) plus event markers and summaries that make instability legible early—before failure propagates.

Inputs
  • Interaction trace: (ut, mt) pairs (text or multimodal references).
  • Optional embeddings: model-agnostic vectors or similarity signals.
  • Optional model-side signals: lightweight load indicators where available (vendor-agnostic).
  • Optional metadata: timing, turn lengths, tool-call boundaries, agent IDs.

H2E works without internals. If internals are available, they can improve Ss estimation, but are not required.

Outputs
  • Telemetry packet: JSON (and/or CBOR) with per-turn metrics + provenance.
  • Trajectory features: trend, volatility, drift onset, collapse flags.
  • Summary report: reproducible text/JSON report for audit pipelines.
  • Visualization hooks: optional adapters for dashboards and overlays.

Downstream: teams may use H2E outputs for monitoring, gating, evaluation studies, or internal optimization loops.
